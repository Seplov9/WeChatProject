## 页面路由与导航

| 页面路径 | tabbar | 说明 |
|----------|--------|------|
| pages/index/index | 是（tab 0） | "信息池"标签，展示所有活跃信息，支持搜索和 6 级筛选，上拉分页加载 |
| pages/post/post | 是（tab 1） | "发布"标签，发布新信息或编辑已有信息，自定义导航栏 |
| pages/profile/profile | 是（tab 2） | "我的"标签，微信登录 + 导航到我的发布/收藏/浏览记录 |
| pages/show/show | 否 | 信息详情页，展示完整信息和图片轮播，右上角收藏星标 |
| pages/publisher/publisher | 否 | 发布者主页，展示发布者信息和其所有发单 |
| pages/order-list/order-list | 否 | 统一列表页，根据 type 参数展示我的发布/收藏/浏览记录 |

- tabbar 采用 `custom: true` 自定义组件实现，代码在 `custom-tab-bar/`
- tab 文案：信息池 / 发布 / 我的
- 每个 tab 页面在 `onShow` 中通过 `this.getTabBar().setData({ selected: N })` 设置选中态
- 非 tab 页面不显示底部导航栏，通过顶部导航栏返回
- **post 页面**使用自定义导航栏（`navigationStyle: "custom"`），编辑模式下显示 `<` 返回按钮
- profile → order-list 使用 `wx.navigateTo`
- order-list（我的发布）→ post 编辑模式：使用 `wx.switchTab` + `app.globalData.pendingEditId` 传参
- index/order-list → show：使用 `wx.navigateTo` + 查询参数 `?orderId=xxx&role=x`
- show → publisher：使用 `wx.navigateTo`，传递 publisherId/name/avatar
- publisher → show：使用 `wx.navigateTo` + 查询参数 `?orderId=xxx`
- post 编辑模式的返回按钮 → `wx.switchTab` 到 profile 页

### 编辑模式进入与退出

1. 从 order-list（我的发布）点击卡片 → 设置 `app.globalData.pendingEditId` + `pendingEditRole` → `wx.switchTab` 到 post
2. post 的 `onShow` 检测 `pendingEditId` → 设置 `_fromCardEdit = true` → `loadOrder(id, role)` 拉取数据
3. 保存或撤销后清除 `_fromCardEdit`
4. 若未保存/撤销就离开（切换 tab），下次 tabbar 进入时 `onShow` 检测到 `_fromCardEdit && editId` → 调用 `resetForm()` 清空
5. 编辑模式下导航栏左上角显示 `<` 返回按钮，点击跳回 profile 页

## 云函数 API（quickstartFunctions）

通过 `event.type` 字段区分操作，共 17 个类型：

| type | 功能 | 参数 | 涉及集合 |
|------|------|------|-----------|
| getOpenId | 获取用户 openid | 无 | - |
| getMiniProgramCode | 生成小程序码 | 无 | - |
| getPhoneNumber | 解密手机号（个体工商户后可用） | event.code | - |
| addUserData | 写入提交数据（含内容安全审核） | event.content | users（type: "submission"） |
| getUserData | 查询所有提交数据（按时间倒序） | 无 | users（type: "submission"） |
| saveUser | 保存/更新用户档案，并同步更新双表中所有该用户订单的昵称和头像 | event.nickName, event.avatarUrl | users + orders_a + orders_b |
| createOrder | 发布信息（role 必选，a=甲方/b=乙方，含内容安全审核） | event.role, category, title, content, reward, contact, city, images, publisherName, publisherAvatar | orders_a 或 orders_b |
| getOrders | 查询信息池（支持角色筛选，双表合并排序，分页） | event.skip, pageSize, roleFilter | orders_a + orders_b |
| getMyOrders | 查询当前用户所有发布（双表合并，按时间倒序） | 无 | orders_a + orders_b |
| getOrderById | 查询单个信息详情 | event.orderId, role | orders_a 或 orders_b |
| updateOrder | 更新信息（验证发布者身份 + 内容安全审核） | event.orderId, role, category, title, content, reward, contact, city, images | orders_a 或 orders_b |
| cancelOrder | 撤销信息（验证发布者身份，设置 status=closed） | event.orderId, role | orders_a 或 orders_b |
| getPublisherOrders | 查询指定发布者的所有有效信息（双表合并） | event.publisherId | orders_a + orders_b |
| toggleFavorite | 切换收藏状态（已收藏则删除，未收藏则新增） | event.orderId, orderSnapshot | favorites |
| getFavorites | 查询当前用户的收藏列表，返回前校验订单仍存在于 orders_a/orders_b | 无 | favorites + orders_a + orders_b |
| recordHistory | 记录浏览历史（同一信息去重：先删后插） | event.orderId, orderSnapshot | history |
| getHistory | 查询当前用户的浏览记录，返回前校验订单仍存在于 orders_a/orders_b（限 50 条） | 无 | history + orders_a + orders_b |

### 收藏与历史的存在校验

- `getFavorites` 和 `getHistory` 在返回数据前，使用 `db.command.in(orderIds)` 批量查询 `orders_a` 和 `orders_b`，校验 `orderId` 是否仍存在
- 已撤销（status=closed）或已删除的订单对应的收藏/历史记录会被过滤掉，不会展示给用户

### 内容安全审核

| 函数 | 审核字段 | scene | 违规返回 |
|------|---------|-------|---------|
| saveUser | nickName | 1（用户资料） | `errMsg: "内容不合规，请修改"` |
| addUserData | inputContent | 2（用户生成内容） | `errMsg: "内容不合规，请修改"` |
| createOrder | title + " " + content | 2（用户生成内容） | `errMsg: "内容包含不合规信息，请修改后重新发布"` |
| updateOrder | title + " " + content | 2（用户生成内容） | `errMsg: "内容包含不合规信息，请修改后重新发布"` |

- `suggest === "pass"` 放行，其余均拦截
- API 调用异常时自动降级放行，不阻塞正常使用
- `config.json` 已声明 `security.msgSecCheck` 权限

## 数据库集合

### orders_a（甲方表 — "我需要"）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| role | string | 固定值 "a"，表示甲方（我需要） |
| publisherId | string | 发布者 openid（云函数服务端取值） |
| publisherName | string | 发布者昵称 |
| publisherAvatar | string | 发布者头像 |
| category | string | 信息分类 |
| title | string | 标题（限 50 字） |
| content | string | 详情（限 500 字） |
| reward | number | 报酬金额（甲方必填，乙方为 0） |
| contact | string | 联系方式 |
| city | string | 城市，格式"省·市·区"，各级均可缺省 |
| images | array | 图片 cloud fileID 数组（最多 9 张） |
| status | string | "active"（进行中）或 "closed"（已结束） |
| boost | number | 推广权重，默认 0 |
| createdAt | date | 发布时间 |
| updatedAt | date | 最近更新时间 |

### orders_b（乙方表 — "我可以"）

字段与 orders_a 相同，`role` 固定值为 "b"。

### 双表设计要点

- 身份选择"我需要"时写入 orders_a，"我可以"时写入 orders_b
- 信息池（getOrders）：可分别查看甲方池/乙方池，也可合并查看（roleFilter 为空时双表合并按 boost + createdAt 排序分页）
- 我的发布（getMyOrders）：双表并发查询，合并后按 createdAt 排序
- 发布者主页（getPublisherOrders）：双表并发查询活跃信息，合并排序
- 用户更新昵称/头像时（saveUser）：并发更新 orders_a 和 orders_b 中该用户所有订单
- 编辑/撤销/详情查询需要传入 role 以定位具体表

### favorites（收藏表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| userId | string | 收藏者 openid |
| orderId | string | 被收藏信息的 _id |
| orderSnapshot | object | 收藏时信息的快照（含 role, category, title, content, reward, contact, city, images） |
| createdAt | date | 收藏时间 |

- 同一用户对同一信息只有一条记录（toggle 逻辑：存在则删除，不存在则新增）
- `orderSnapshot` 保存快照，包含 `role` 字段用于回查原表
- 查询时通过 `db.command.in` 批量校验 orderId 是否仍存在于 orders_a/orders_b

### history（浏览记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | string | 自动生成 |
| userId | string | 浏览者 openid |
| orderId | string | 被浏览信息的 _id |
| orderSnapshot | object | 浏览时信息的快照（含 role, category, title, content, reward, contact, city, images） |
| viewedAt | date | 浏览时间 |

- 同一用户浏览同一信息多次，只保留最新一条（去重：先删旧记录，再插入新记录）
- `orderSnapshot` 保存快照，与 favorites 设计一致
- 查询时通过 `db.command.in` 批量校验 orderId 是否仍存在于 orders_a/orders_b

### users（用户表）

存放两类文档：

**用户档案**（saveUser 写入）：以 openid 为唯一标识，含 nickName、avatarUrl、预留 phoneNumber/memberType/memberExpire 字段。

**提交记录**（addUserData 写入，旧 demo 遗留）：type 固定为 "submission"，含 inputTime 和 inputContent。

## 数据库复合索引

| 集合 | 索引字段 | 用途 |
|------|---------|------|
| orders_a | status + boost（降序）+ createdAt（降序） | getOrders 查询：where status=active, orderBy boost desc, createdAt desc |
| orders_a | publisherId + createdAt（降序） | getMyOrders / getPublisherOrders：where publisherId=xxx, orderBy createdAt desc |
| orders_b | status + boost（降序）+ createdAt（降序） | getOrders 查询 |
| orders_b | publisherId + createdAt（降序） | getMyOrders / getPublisherOrders |
| favorites | userId + createdAt（降序） | getFavorites 查询 |
| history | userId + viewedAt（降序） | getHistory 查询 |

共 6 个复合索引，需在微信云开发控制台手动创建。

## 页面功能详情

### 信息池（pages/index/index）

**布局**：
- 顶部搜索栏：白色圆角输入框（72rpx 高，36rpx 圆角），placeholder "搜索信息..."
- 两行筛选区（`justify-content: space-between`，6 格等宽 216rpx 对齐）：
  - 第一行：身份 / 线上/线下 / 分类
  - 第二行：省 / 市 / 区
- 每个 picker 样式统一：60rpx 高，白色背景，8rpx 圆角，24rpx 字号
- 卡片列表：占满页面宽度，卡片间 4rpx 灰色间隔
- 底部加载提示：加载中 / 上拉加载更多 / 已加载全部

**交互逻辑**：
- 身份（roleIndex）：默认"身份"不筛选；选择"“我可以”"展示甲方池（roleFilter="a"）；选择"“我需要”"展示乙方池（roleFilter="b"）；切换身份时重新请求数据
- 线上/线下：默认"线上/线下"不筛选；选择"线上"仅显示线上分类；选择"线下"仅显示线下分类；切换时重置分类/省/市/区
- 分类：默认"分类"不筛选；线上/线下切换时联动更新分类选项
- 省→市→区三级级联：省默认"全部"，市和区默认"全部"；选择省后级联更新市列表，选择市后级联更新区列表；直辖市省份下拉框中有该直辖市名，选择后市下拉框仅显示该直辖市名，区下拉框显示真实区名
- 搜索：输入文字后回车触发，按标题和内容双字段 `includes()` 匹配（任一命中即展示）
- 筛选均为客户端即时过滤（不对数据库重新请求，角色切换除外）
- 收藏按钮：空心五角星 ☆ / 实心 ★，点击调用 toggleFavorite
- 点击卡片：记录浏览历史 → navigateTo 详情页（带 role 参数）
- 上拉触底：加载下一页（每页 10 条），直到 hasMore 为 false
- 下拉刷新：触顶下滑触发 `onPullDownRefresh`，重新加载数据
- onShow 时自动拉取收藏状态和信息列表，重置所有下拉框为默认值

**关键样式参数**：
- 容器底部留白：`calc(100rpx + env(safe-area-inset-bottom) + 20rpx)`
- 卡片图片：宽 100%，高 280rpx，圆角 8rpx
- 报酬：红色 #ee3f4d，36rpx 加粗，仅 reward > 0 时显示
- 收藏按钮：48rpx × 48rpx，字号 36rpx，未收藏灰色 #ddd，已收藏金色 #f5a623

### 发布（pages/post/post）

**布局**：
- 自定义导航栏：白色背景 + 状态栏自适应高度，44px 导航内容区，居中标题"发布"
- 编辑模式下导航栏左侧显示 `<` 返回按钮
- 头部：📋 图标 + "发布信息"标题 + "让更多人看到你的需要或能力"
- 信息分类区（白色圆角 section）：
  - 两行下拉筛选（6 格等宽，与信息池样式统一）：身份 / 线上/线下 / 分类 + 省/市/区
  - 所有下拉框均无"全部"选项，使用占位文字（"身份"/"线上/线下"/"分类"/"省份"/"市/区"/"区"）
- 图片区（白色 section，可选，最多 9 张）：144rpx 方形缩略图网格，虚线边框"+"添加按钮
- 信息详情区（白色 section）：标题输入框（限 50 字，带字数计数）+ 详情文本域（限 500 字，带字数计数）
- 报酬与联系区（白色 section）：
  - 当身份选择"“我需要”"时：报酬（¥ 前缀）+ 联系方式，双列并排
  - 当身份选择"“我可以”"时：仅联系方式，占满宽度
  - section 标题动态切换："报酬与联系" / "联系方式"
- 提交区：
  - 新建模式：三按钮一行 — "立即发布"（绿色渐变，占 50%）+ "保存"（绿色描边）+"重置"（灰色描边）
  - 编辑模式：两按钮一行 — "保存修改"（绿色渐变）+ "撤销发布"（红色描边）

**交互逻辑**：
- 身份必选（roleIndex !== 0），否则提示"请选择身份"
- 线上/线下必选（onlineIndex !== 0），否则提示"请选择线上/线下"
- 分类必选（categoryIndex !== 0），否则提示"请选择品类"
- 标题、内容、联系方式均不可为空
- 选择"“我需要”"时薪酬必填（> 0），选择"“我可以”"时薪酬强制为 0
- 城市级联校验：省必选；直辖市时区必选，普通省时市必选、区有数据时必选
- city 字段构造："省" + (如果选了市且市名≠省名则 "·市") + (如果选了区则 "·区")
- 直辖市（北京/天津/上海/重庆）：下拉5=直辖市名（唯一选项，自动选中），下拉6=真实区列表
- 图片上传：`wx.chooseMedia`（压缩）→ `wx.cloud.uploadFile` → 存储 fileID
- 身份映射："“我可以”"→role="b"(写入orders_b)，"“我需要”"→role="a"(写入orders_a)
- 编辑模式：从我的发布进入，表单回填原有数据，保持原 role 写入原表
- **保存**：将当前表单状态写入 `wx.Storage`（key: "postDraft"），下次进入时自动恢复
- **重置**：清除 draft 并重置所有表单字段
- 发布成功后清除 draft
- 编辑模式下点击 `<` 返回按钮 → `wx.switchTab` 到 profile 页

**关键样式参数**：
- 页面底部留白：`calc(100rpx + env(safe-area-inset-bottom) + 40rpx)`
- 提交按钮：96rpx 高，48rpx 圆角，渐变绿 + 阴影
- 撤销按钮：96rpx 高，白色 + 红色边框
- 导航栏：`position: sticky; top: 0; z-index: 100`，返回箭头 `<` 加粗 18px

### 个人信息（pages/profile/profile）

**未登录状态**：
- 顶部白底圆角 box（140rpx 高），左侧灰色默认头像 + 右侧"请登录"
- 点击弹出登录弹窗

**登录弹窗**：
- 标题"欢迎登录"
- `<button open-type="chooseAvatar">` 获取头像
- `<input type="nickname">` 获取昵称
- "请您同意"勾选框，未勾选时按钮禁用
- 调用 `saveUser` 写入用户档案，同步更新所有订单中的昵称和头像
- 登录状态通过 `wx.setStorageSync` 本地持久化

**登录后状态**：
- 真实头像 + 昵称 + "退出登录"按钮
- 三个导航入口：我的发布 / 我的收藏 / 浏览记录，跳转至 order-list

**修改资料弹窗**（已登录时点击 box 弹出）：预填当前头像和昵称，可更换后保存。

**关键样式参数**：
- 弹窗 mask：z-index 9999
- 弹窗 popup：z-index 10000，底部留白防止 tabbar 遮挡

### 统一列表页（pages/order-list/order-list）

接收 `?type=myOrders|favorites|history` 参数，动态切换数据源和标题。

**布局**：

### 全国省市数据（data/regions.js）

## 用户强调过的要求

### 登录方式
- 当前使用 `open-type="chooseAvatar"` 获取头像 + `<input type="nickname">` 获取昵称
- 旧的 `open-type="getUserInfo"` 和 `wx.getUserProfile` 已失效，不可用
- 手机号快捷登录仅对企业/个体工商户开放，个人主体暂无权限

### 登录持久化
- 登录成功后 `wx.setStorageSync` 保存，`onLoad` 时恢复
- 退出登录时 `wx.removeStorageSync` 清除

### 样式原则
- 多个页面共用的 UI 组件（筛选区、卡片、搜索栏）样式必须统一
- 6 个筛选下拉框等宽（216rpx），`justify-content: space-between` 对齐
- 筛选下拉框统一：60rpx 高，白色背景，8rpx 圆角，24rpx 字号
- 按钮文字居中：`display: flex; align-items: center; justify-content: center; padding: 0;`
- 卡片占满页面宽度，卡片间用细缝分隔
- 弹窗需避开自定义 tabbar，层级高于 tabbar

### 筛选与导航
- 信息池和列表页的搜索+筛选功能完全一致
- 搜索范围为标题+内容双字段匹配
- 下拉框默认值均为占位符：身份→"身份"、线上/线下→"线上/线下"、分类→"分类"、省→"全部"/"省份"（post 页）、市→"全部"/"市/区"（post 页）、区→"全部"/"区"（post 页）
- 编辑模式：从我的发布进入发布页，未保存就离开则下次 tabbar 进入清空表单；手动填写的表单可通过"保存"按钮保留草稿
- 身份文案使用中文引号："“我可以”" 和 "“我需要”"

### 数据库设计
- 全部用户数据通过 openid（云函数服务端取值）关联，无法伪造
- 信息按甲方（我需要）/ 乙方（我可以）分表存储（orders_a / orders_b）
- 收藏和浏览记录保存 orderSnapshot 快照（含 role 字段），防止原信息删除后数据丢失
- saveUser 更新用户信息时同步更新双表中该用户所有订单的昵称和头像
- 收藏和历史记录查询时校验引用订单是否仍存在于 orders_a/orders_b，过滤已撤销/不存在的记录

### 开发习惯
- 修改代码聚焦在用户真正提出的问题上
- 同一个问题出现在多个页面时，一并修复

### 云函数
- 本地调试时操作的是真实云端数据库
- 上传云函数推荐用"云端安装依赖"
